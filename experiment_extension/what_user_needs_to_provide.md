# What the Author Needs to Provide

Most of the upgrade can be prepared locally. The author only needs to provide the following decisions or external inputs.

## Required

1. Decide whether to pursue:
   - strong special issue / journal route; or
   - 2027 strong conference route; or
   - safer journal route.

2. Choose how to generate repeated-run and ablation outputs:
   - use ChatGPT/Claude/Gemini manually and paste/export CSV outputs; or
   - use an API if available; or
   - use Codex only for protocol preparation and wait for manual generation.

3. Provide model details for any generated outputs:
   - model name;
   - provider;
   - date;
   - temperature/top-p if known;
   - whether outputs were edited after generation.

## Highly recommended

4. Ask one mechanical/reliability teacher, classmate, or engineer for a 20-30 minute blinded plausibility review.

5. Allow the reproducibility package to be deposited publicly through OSF, Zenodo, GitHub, or another repository.

## Optional

6. Provide any original generation logs or screenshots if they exist.

7. Provide a preferred target venue so the paper can be formatted to the right length and style.

## Not needed

- No physical equipment is needed.
- No laboratory testing is needed for the next upgrade.
- No paid AI-detection service is needed.
- No claim of safety certification should be added.
